Jenkins-忘记admin或其他用户密码的解决办法

发表于 2018-04-28 | 分类于 工具 ， Jenkins


Linux环境中，Jenkins忘记密码的情形分好几种：

admin无法登录

一种情况是安装了jenkins后，没有立即用admin登录，用的是自己创建的用户，第二次登录时就发现 admin 登录不了（即使默认的初始密码你已经拷贝出来了）。这是因为 jenkins 安装目录下的 users 文件夹中，根本就没有 admin 这个账户。

另外一种情况是：你真的忘记了 admin 的密码。

不用着急，先确定自己是哪种情况，去 jenkins 默认的安装目录找找看有没有 admin 账户。
在 users 目录下，如果没有 admin 文件夹，就表示没有 admin 这个账户。

一般情况下，如果是通过 rpm 包安装的 jenkins，其默认路径如下：
/usr/lib/jenkins/jenkins.war WAR包
/etc/sysconfig/jenkins 配置文件
/var/lib/jenkins/ 默认的JENKINS_HOME目录
/var/log/jenkins/jenkins.log Jenkins日志文件

其中，users 文件夹在 /var/lib/jenkins/ 目录下。如果是通过其他方式装的，或者自己指定了 jenkins 安装目录，就到自己的安装目录下去找。

没有admin账户的解决办法

如果你的 users 目录下有其他账户，例如 zebe，可以通过 cp -r zebe admin 命令，拷贝一个账户并命名为 admin，这样你就有了一个 admin 账户，密码和 zebe 一样。前提是你记得 zebe 这个账户的密码。

有admin账户，但是忘记了密码的解决办法

其实，仔细查看用户目录里面的 config.xml，就会发现里面保存了用户的密码，只不过是通过 Hash 加密了（请留意 passwordHash 这个节点里面的内容）。把其他账户的配置文件里面的密码，拷贝到 admin 账户的 config.xml 中，就行了，密码和被拷贝的那个账户的密码相同。

既没有admin账户，其他账户的密码也忘记了解决办法

使新的账户和密码生效

先停止jenkins，执行命令：service jenkins stop。
再启动jenkins，执行命令：service jenkins start。

如果没有 jenkins 服务的话（通过 rpm 装的都有这个服务），请以其他的方式重启 jenkins。
