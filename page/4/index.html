发送HTTP范围请求获取图像的部分内容

发表于 2018-03-29 | 分类于 协议 ， HTTP

什么是范围请求

以前，用户不能使用现在这种高速的带宽访问互联网，当时，下载一个尺寸稍大的图片或文件就已经很吃力了。如果下载过程中遇到网络中断的情况，那就必须重头开始。为了解决上述问题，需要一种可恢复的机制。所谓恢复是指能从之前下载中断处恢复下载。

要实现该功能需要指定下载的实体范围。像这样，指定范围发送的请求叫做范围请求（Range Request）。

对一份 10000 字节大小的资源，如果使用范围请求，可以只请求5001~10000 字节内的资源。

如下图所示：

判断服务器是否支持范围请求

发送一个普通 GET 或者 HEAD 请求，如果服务器的响应首部字段中，包含 Accept-Ranges: bytes，则说明服务器支持范围请求。

请求方法

服务器首部字段中的 Content-Length 指明了某个请求内容的大小（字节）。
在HTTP请求首部字段中加入 Range 参数，指定值例如：bytes=0-5200，如果一张图片有98757字节大小，则该请求表示获取总长度为 5201 个字节范围的数据。

SpringBoot关键知识点简要笔记

发表于 2018-03-22 | 分类于 框架 ， SpringBoot

内容简介

本文是个人对 SpringBoot 关键知识点的梳理和简单总结，方便遗忘时回顾。文章内容边学边整理。

关于参数、配置文件(profile)

命令行参数是最高加载级别的参数，通过两个连字符指定，如：java -jar demo.jar –server.port=8888。

要添加多个配置文件（profile），添加多个 application-xxx.properties 配置即可，如：

  • application-dev.properties
  • application-test.properties

然后在 application.properties 中只需要写一行 spring.profiles.active=dev，即表示使用-dev结尾的这个配置文件。

YAML格式的配置文件，其数据加载到内存中时，数据是有序的。

配置文件中使用随机数：

  • ${random.value}：随机字符串
  • ${random.int}：随机int
  • ${random.long}：随机long
  • ${random.int[10]}：10以内的随机整数
  • ${random.int[10,20]}：10~20以内的随机整数

在配置文件中写入以 info 为前缀的配置，则可用通过 actuator 组件的 /info 请求来获取这些信息，如果不设置，则默认返回的是空集。

关于actuator组件

要使用 actuator，必须同时引用 starter-web，至少对于1.3.7版本是这样。

在发现一些配置没有生效时，可以通过 actuator 查看项目的自动配置报告，访问 /autoconfig，找到 negativeMatches 节点的内容进行分析。

查看所有已创建的bean，可以 访问 /beans，每个bean节点里的 dependencies 列出了该bean的所有依赖。

查看所有可用的环境变量、JVM属性、应用配置属性等等，访问 /env。

查看所有的控制器映射，访问 /mappings，即可查看所有的请求映射。

查看应用的所有动态指标，访问 /metrics，包括内存、线程、类、HTTP请求统计等动态数据。如果发现某个请求特别慢，可以检查 gauge.response.xxx 这类的数据，数字是上一次请求的毫秒延迟时间(ms)，如果数字较大，证明该请求延迟较高。

查看应用的健康状态，访问 /health，可以自己实现 HealthIndicator 接口来执行自己的健康检测，例如 MyCustomHealthIndicator，如果以这个为类的名称，那么健康信息中返回的节点就是 myCustom。

查看应用的线程信息，访问 /dump，特别是检查线程是否阻塞、死锁等情况非常有用。

要远程关闭应用，访问 /shutdown，前提是配置中指定了 endpoints.shutdown.enabled=true。
