MySQL5.6自定义data文件夹路径后无法启动发生系统错误1067 - 解决办法

发表于 2018-06-08 | 分类于 数据库 ， MySQL


这篇文章记录一下Windows下MySQL服务无法启动系统错误1067的原因和解决办法。问题很简单，记录很详细，这样有助于提高自己的思维和排查问题的能力。

启动MySQL5.6.40服务出错

服务成功安装后，我又编写了一个启动脚本 MySQL-5.6.40一健启动.bat，方便将来手动一键启动。内容很简单，如下所示：

这个脚本无法成功执行，当我在命令提示符中直接粘贴运行 net start MySQL5.6.40 这句命令时，提示 MySQL5.6.40 服务无法启动、发生系统错误 1067。

原因分析及解决

打开 Windows事件查看器，找到 Windows日志，在应用程序日志列表中，我发现MySQL有一个致命错误：Fatal error: Can’t open and lock privilege tables: Table ‘mysql.user’ doesn’t exist。

既然说是找不到默认的mysql数据库中的user表，那么我去MySQL的安装目录下的data文件中，发现有user这个表。
于是，我把整个 data 文件夹里面的内容，全部复制粘贴到我在 my.ini 配置文件中指定的目录：D:\database\mysql（注意是直接整个目录下粘贴原来data文件夹下面的所有内容，不要把data也作为目录粘贴进去，因为我们指定的目录，就是MySQL所需的data文件夹），然后再启动服务，服务即可成功启动。
